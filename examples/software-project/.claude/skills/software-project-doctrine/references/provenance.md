# Provenance — Acme SaaS

## L1.1 — Trustworthy first, fast second

- **Originated:** 2026-01-18
- **By:** Team offsite, founder-led
- **Context:** Early user interview round revealed compliance/audit use case was the anchoring value. Speed was secondary and sometimes actively harmful.
- **Evidence:** `docs/product/founding-narrative.md`, `docs/product/ops-user-interview-2026-Q1.md`
- **Status:** active

## L1.2 — AI outputs are proposals, not decisions

- **Originated:** 2026-01-30
- **By:** Rollback decision after early auto-apply prototype
- **Context:** Pre-launch prototype auto-applied AI suggestions. Three pilot users lost hours undoing. We concluded AI cannot mutate user data without explicit user action.
- **Evidence:** `docs/retrospectives/2026-01-autoapply-rollback.md`
- **Status:** active

## L1.3 — Every AI answer must be citable

- **Originated:** 2026-03-15
- **By:** Q&A launch decision
- **Context:** Users tested the Q&A surface and consistently asked "where did you get that?" after every answer. We concluded the citation was not a nice-to-have.
- **Evidence:** `docs/product/qna-user-testing-2026-03.md`
- **Status:** active

## L1.4 — Kill-switch before kill

- **Originated:** 2026-04-10
- **By:** Post-incident review (2026-04-08 vendor refusal event)
- **Context:** Kill-switch existed but had never been exercised under load. When needed, its cascade broke things. We concluded exercise is required before shipping.
- **Evidence:** `docs/retrospectives/2026-04-08-vendor-refusal-incident.md`
- **Status:** active
